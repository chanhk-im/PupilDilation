## 입력받을 정보
- 작업 제목: $ARGUMENTS

## 구현 단계 DB
- data source ID: `33a0e39821a6804097fefffc10b50740`
- 상태 값: "시작 전" / "진행 중" / "완료"
- 페이지 내부: 각 단계 상세 내용 체크리스트
- 작업 시작 시 해당 단계를 "진행 중"으로 업데이트
- 작업 완료 시 "완료"로 업데이트

## 핵심 컨벤션
- 컨벤션은 노션 코드 컨벤션 페이지를 반드시 읽고 적용할 것
- 코드 컨벤션 페이지: `32d0e39821a680adbf60d7a8538d71a2`

## 1단계 — GitHub 이슈 생성
GitHub MCP로 이슈를 생성해줘.
- 참고자료: 설계문서(3290e39821a680129e60df8280c6a06d)
- 레포: chanhk-im/PupilDilation
- 제목: [FEAT/FIX/...] 입력받은 작업 제목
- 내용: 작업 상세 내용 체크리스트
- 라벨: feat, fix, refactor, docs, test, ... (작업 성격에 맞게 판단)

## 2단계 — 노션 개발일지 생성
Notion MCP로 개발일지 DB(32d0e39821a6802daf88fb653f9507b7)에 새 항목을 추가해줘.
- 이름: 작업 제목(대상 + 행위) ex) AuthController 구현
- issue: feat/#{이슈번호} fix/#{이슈번호} ...
- 내용: 작업 상세 내용 체크리스트

## 3단계 — 브랜치 생성
아래 네이밍 규칙으로 브랜치를 생성하고 체크아웃해줘.
- 브랜치 생성 base: dev
- 기능: feat/#{이슈번호}-{작업명-kebab-case}
- 버그: fix/#{이슈번호}-{작업명-kebab-case}
- 리팩토링: refactor/#{이슈번호}-{작업명-kebab-case}
