# 작업 마무리

현재 브랜치 기준으로 아래 순서대로 진행해줘.
각 단계 완료 후 결과를 보고하고 다음 단계로 넘어가.

## 1단계 — 코드 리뷰
변경된 파일 목록을 확인하고 컨벤션 기준으로 리뷰해줘.
컨벤션 페이지 ID: 32d0e39821a680adbf60d7a8538d71a2
문제가 있으면 리뷰 결과를 보고하고 계속 진행할지 물어봐줘.

## 2단계 — 커밋 & 푸시
- 커밋 메시지 형식: feat[#{이슈번호}]: {작업 내용 요약}
- 버그픽스는 fix, 리팩토링은 refactor 사용
- 변경 파일 목록 확인 후 커밋 메시지 제안 → 승인받고 실행

## 3단계 — PR 생성
GitHub MCP로 PR을 생성해줘.
- 레포: chanhk-im/PupilDilation
- 제목: feat: #{이슈번호} 작업 제목
- 본문:
  ## 개요
  {변경 사항 요약}

  ## 변경 파일
  {변경된 파일 목록}

  ## 리뷰 포인트
  {리뷰어가 집중해서 볼 부분}

  Closes #{이슈번호}

## 4단계 — 노션 개발일지 업데이트
해당 이슈 번호의 개발일지 항목에 PR URL과 완성일자를 업데이트해줘.
- 개발일지 DB: 32d0e39821a6802daf88fb653f9507b7

## 5단계 — Merge 확인
PR URL을 보여주고 "merge할까요?" 라고 물어봐줘.
승인 시 squash merge 실행.

## 6단계 = 구현 단계 DB
- data source ID: 33a0e398-21a6-80d8-9cec-000b38ed1c5a
- 상태 값: "시작 전" / "진행 중" / "완료"
- 페이지 내부: 각 단계 상세 내용 체크리스트
- 작업 시작 시 해당 단계를 "진행 중"으로 업데이트
- 작업 완료 시 "완료"로 업데이트
- 해당 단계의 페이지에 상세 내용 업데이트
